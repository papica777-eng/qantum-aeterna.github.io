QAntum Prime Docs

The complete reference for the Autonomous QA Engine — from first install to enterprise-scale deployment. No fluff. Just production-ready patterns.

Zero-Config Start

One command. Runs your first test in under 60 seconds.

Self-Healing

Swarm Execution

1,247 concurrent workers. 5 global regions. Infinite scale.

Predictive Intelligence

Chronos forecasts failures 48h before they occur.

Installation

Install via npm, yarn, or pnpm. Requires Node.js ≥ 18.0.

bash 
# npm
npm install qantum-prime

# yarn
yarn add qantum-prime

# pnpm
pnpm add qantum-prime

Scaffold a new project

bash 
npx qantum init my-project
cd my-project
npm test
Browser engines auto-installed

QAntum automatically downloads Chromium, Firefox, and WebKit during first run via its built-in browser provisioner. No manual Playwright/Selenium setup needed.

Quick Start

typescript 
import { QAntum } from 'qantum-prime';

// Initialize the engine — all options are optional
const qa = await QAntum.create({
  ghost:    true,   // Invisible bot-proof execution
  selfHeal: true,   // Auto-repair broken selectors
  parallel: 10,    // Concurrent test workers
});

// Define your test
await qa.test('Homepage loads correctly', async ({ page }) => {
  await page.goto('https://your-app.com');
  await qa.expect(page).toHaveTitle(/Your App/);
  await qa.expect(page.locator('[data-testid="hero"]')).toBeVisible();
});

// Execute
await qa.run();
That's your first test.

QAntum handles browser provisioning, self-healing, parallel scheduling, and HTML reporting automatically. No extra config needed for day one.

Configuration

Create qantum.config.ts in your project root for project-wide settings:

typescript — qantum.config.ts 
import { defineConfig } from 'qantum-prime';

export default defineConfig({

  // ── Ghost Protocol ──────────────────────────────
  ghost: {
    enabled:     true,
    biometric:   true,       // Human mouse curves + typing variance
    fingerprint: 'unique',  // Fresh browser identity per session
    stealth:     'maximum', // Bypass Cloudflare, Akamai, DataDome
  },

  // ── Self-Healing ─────────────────────────────────
  selfHeal: {
    enabled:             true,
    strategies:          ['data-testid', 'aria-label', 'text', 'visual'],
    confidenceThreshold: 0.85,  // Min ML confidence to auto-apply
    logHealing:          true,  // Export healing events to report
  },

  // ── Parallel Execution ───────────────────────────
  parallel: {
    workers: 50,
    retries: 2,
    timeout: 30000,  // ms — per test
  },

  // ── Browsers ─────────────────────────────────────
  browsers: ['chromium', 'firefox', 'webkit'],

  // ── Reporting ─────────────────────────────────────
  reporters: ['html', 'json', 'console'],
  outputDir: './test-results',

  // ── Swarm (Enterprise) ───────────────────────────
  swarm: {
    enabled:  false,
    workers:  1000,
    regions:  ['us-east', 'eu-west', 'ap-south'],
  },

});

Ghost Protocol v2

Ghost Protocol makes QAntum completely invisible to bot-detection layers — Cloudflare, Akamai, PerimeterX, DataDome — by simulating statistically human behavior at the hardware and browser-API level.

Capabilities

  • Biometric Simulation — Realistic Bézier-curve mouse paths, typing velocity variance (±40ms), and randomized scroll momentum.
  • Fingerprint Masking — Unique navigator object, Canvas fingerprint, WebGL renderer, and audio context per session.
  • Anti-Detection Suite — Patches navigator.webdriver, window.chrome, and all headless browser tells.
  • TLS Fingerprint Rotation — JA3/JA4 fingerprint matching real Chrome/Firefox builds.
typescript 
const qa = await QAntum.create({
  ghost: {
    enabled:     true,
    biometric:   true,
    fingerprint: 'unique',  // 'unique' | 'consistent' | 'rotating'
    stealth:     'maximum', // 'standard' | 'maximum'
    proxy: {
      enabled:  true,
      type:     'residential',
      rotation: 'per-session',
    },
  },
});

Self-Healing Engine

When a selector breaks because UI changed, QAntum's ML engine silently finds the correct element and continues the test — then logs exactly what it healed so you can update your source.

Healing Pipeline

  1. Broken selector detected — test pauses without crashing.
  2. DOM snapshot taken and encoded into the ML feature vector.
  3. 15 alternative strategies scored in parallel (avg <12ms).
  4. Highest-confidence match applied if score ≥ confidenceThreshold.
  5. Test continues. Healing event logged to test-results/healing.json.

Selector Strategy Priority

PriorityStrategyAccuracyDescription
1data-testid99.2%Explicit test attribute — highest signal, lowest noise
2aria-label94.7%Semantic accessibility label — stable across style refactors
3role + name93.1%ARIA role combined with accessible name
4text-content91.3%Visible text as locator — good for CTAs and navigation
5placeholder89.4%Input placeholder text for form elements
6visual-position87.8%Spatial relationship (above/below/next-to) other stable elements
7dom-structure82.4%CSS selector regenerated from DOM path heuristics
8+8 more…≥73%Visual embedding, XPath fallbacks, shadow DOM traversal

Swarm Execution

Swarm distributes your entire test suite across 1,247 concurrent cloud workers in 5 global regions, reducing a 4-hour regression suite to under 4 minutes.

typescript 
const qa = await QAntum.create({
  swarm: {
    enabled:        true,
    workers:        1000,
    regions:        ['us-east-1', 'eu-west-1', 'ap-south-1'],
    loadBalancing:  'intelligent', // Latency-aware routing
    failover:       'hot-standby', // <50ms worker replacement
  },
});

// Watch real-time swarm telemetry
qa.swarm.on('worker:failover', ({ workerId, latencyMs }) => {
  console.log(`Hot-standby activated: ${latencyMs}ms`);
});

The Oracle — Auto-Discovery

Point The Oracle at any URL. It crawls the app, maps every user flow, and generates a complete, ready-to-run test suite. One command replaces weeks of manual test writing.

typescript 
await qa.oracle.discover({
  url:           'https://your-app.com',
  depth:         3,         // Crawl depth (pages)
  generateTests: true,     // Write .spec.ts files to ./tests/
  coverage:      ['forms', 'navigation', 'api', 'auth'],
  auth: {
    type:     'credentials',
    username: process.env.TEST_USER,
    password: process.env.TEST_PASS,
  },
});
// Output: ./tests/ populated with spec files per route

Chronos Engine — Predictive QA

Chronos analyzes historical run data, code commit diffs, and deployment frequency to predict which tests will fail in the next 48 hours — before they do.

typescript 
const predictions = await qa.chronos.predict({ horizon: '48h' });

predictions.forEach((p) => {
  console.log(p.test, p.failureProbability + '%', p.recommendation);
});

// Example output:
// "checkout.spec.ts"  87%  "Payment selector stale — recent DOM change detected"
// "auth.spec.ts"      12%  "Low risk — stable"

Fortress — Security Scanning

Run OWASP-class security scans as part of every test run. Fortress discovers XSS, SQLi, CSRF, SSRF, and auth-bypass vulnerabilities in the same pipeline as your functional tests.

typescript 
const qa = await QAntum.create({
  fortress: {
    enabled:              true,
    scanTypes:            ['xss', 'sqli', 'csrf', 'ssrf', 'auth-bypass'],
    reportVulnerabilities: true,
    failOnSeverity:        'high', // 'low' | 'medium' | 'high' | 'critical'
  },
});

Guide: Testing Login Flows

typescript — tests/auth.spec.ts 
import { QAntum, test, expect } from 'qantum-prime';

// ── Shared auth state — reuse across suite ──────────────
let authState: AuthState;

test.beforeAll(async ({ page }) => {
  authState = await QAntum.auth.login({
    url:      'https://your-app.com/login',
    username: process.env.TEST_USER!,
    password: process.env.TEST_PASS!,
    saveAs:   'admin',  // Reuse in subsequent tests
  });
});

test('Login — valid credentials', async ({ page }) => {
  await page.goto('https://your-app.com/login');
  await page.fill('[data-testid="email"]',    'user@example.com');
  await page.fill('[data-testid="password"]', 'correct-password');
  await page.click('[data-testid="login-btn"]');
  await expect(page).toHaveURL(/\/dashboard/);
});

test('Login — invalid credentials shows error', async ({ page }) => {
  await page.goto('https://your-app.com/login');
  await page.fill('[data-testid="email"]',    'user@example.com');
  await page.fill('[data-testid="password"]', 'wrong-password');
  await page.click('[data-testid="login-btn"]');
  await expect(page.locator('[data-testid="error-msg"]')).toBeVisible();
});

Guide: API Interception & Mocking

typescript 
test('Shows error toast on API 500', async ({ page }) => {
  // Intercept and mock before navigation
  await page.route('**/api/users', (route) =>
    route.fulfill({
      status:      500,
      contentType: 'application/json',
      body:        JSON.stringify({ error: 'Internal Server Error' }),
    })
  );

  await page.goto('https://your-app.com/users');
  await expect(page.locator('[data-testid="error-toast"]')).toBeVisible();
});

// Spy on requests without blocking them
const requests: string[] = [];
page.on('request', (req) => {
  if (req.url().includes('/api/')) requests.push(req.url());
});

Guide: Auth State Management

Use storageState to reuse authenticated sessions across your entire suite — eliminating repeated login steps and saving significant execution time.

typescript — global-setup.ts 
import { QAntum } from 'qantum-prime';
import path from 'path';

async function globalSetup() {
  const browser = await QAntum.launch();
  const page = await browser.newPage();

  await page.goto('https://your-app.com/login');
  await page.fill('[data-testid="email"]', process.env.TEST_USER!);
  await page.fill('[data-testid="password"]', process.env.TEST_PASS!);
  await page.click('[data-testid="login-btn"]');
  await page.waitForURL('**/dashboard');

  // Persist cookies + localStorage to disk
  await page.context().storageState({
    path: path.join(__dirname, 'auth.json'),
  });

  await browser.close();
}

export default globalSetup;

Core API

QAntum.create(options)

static create(options: QAntumOptions): Promise<QAntumInstance>

Creates and initializes a new QAntum engine instance. Always await this call.

Page Object API

MethodSignatureDescription
page.goto()url, opts?Navigate to URL. Waits for load event by default.
page.click()selector, opts?Click element. Auto-waits for visibility + stability.
page.fill()selector, valueClear and type into an input. Triggers input + change events.
page.type()selector, text, opts?Type character-by-character with realistic timing.
page.locator()selectorReturns a lazy locator — query is deferred until action.
page.waitForSelector()selector, opts?Explicit wait for element to match state. Never use fixed sleeps.
page.waitForURL()url | regexpWait for navigation to match URL pattern.
page.route()pattern, handlerIntercept and mock network requests.
page.screenshot()opts?Capture full page or element screenshot.
page.evaluate()fn, ...argsExecute JavaScript in browser context.
